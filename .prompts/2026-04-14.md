## Task 5: Act on test plan review findings

Derived from Task 4 review. Priority ordered. Each step should be a separate branch with a code-review agent per CLAUDE.md.

1. **Ban tautological assertions by policy.** `check(..., true, ...)`, `|| true`, `check(..., a==b || a!=b, ...)` must fail lint/review. Add a CI grep or a pre-commit hook that rejects these patterns in `test/`.

2. **Fix cheap, high-impact plan/test-code bugs first** (no emulator work; one session each). Expected pass-rate gains are the Task 4 estimates, not guarantees:
   - DMA: global replace `0x2D → 0xAD` for R4 in `test/dma/dma_test.cpp` (~23 sites). Introduce named constants for all R0..R5 base bytes. Expected: +28 passing.
   - UART/I2C: fix I2C ACK sampling phase in the test harness (`test/uart/uart_test.cpp` `i2c_send_byte` / `i2c_read_byte`) — sample SDA after the SCL falling edge, not on the rising edge. Expected: +8 passing.
   - Audio: fix envelope shapes 11 and 15 (glyphs inverted in plan) and AY-117 expected value (YM mode should expect 0xE0, not 0xFF). Investigate the real Turbosound default-panning emulator bug separately.
   - ULA Video: fix 4 hand-computed constants in plan + test: S01.09 (`0x0890`/`0x1990`), S01.11 (`0x0900`/`0x1900`), S02.02 (`0x10`), S02.04 (`0x1C`). Remove `|| true` placeholders in S13.03/06/09.
   - MMU: fix SPE-04 expected row to `{0x08,0x09,0x0E,0x0F,0x0C,0x0D,0x06,0x07}` in both plan and test (VHDL is source of truth — +3 config 3 gives 16K banks `{4,7,6,3}`). Apply the RST-01/02 emulator fix (don't clobber `slots_[0..1]` in `Mmu::reset()`).
   - NextREG: decide where VHDL reset defaults live. Recommended: move them into `NextReg::reset()` directly so the bare class owns them (matches VHDL state-process semantics). Apply once, expect the 14 failures to collapse.

3. **Fix the real Tilemap emulator bug**: control-bit mapping swap in `Tilemap::set_control` (VHDL has bit 3 = text mode, bit 4 = palette select, bit 5 = strip_flags; the C++ class has bit 5 = text_mode_, bit 4 = force_attr_). Delete the anti-tests that document this bug as expected (TM-CB5 etc.) and rewrite them to assert VHDL-correct behaviour.

4. **Discuss MMU L2M-01 separately with the user.** The test asserts L2 writes to bank 8 shouldn't alias MMU page 0x10, but VHDL confirms the collision is real (same SRAM page). Options: rewrite test to decouple L2 base from MMU page under test, add a new "known hardware collision" negative test, or delete. Do not touch until agreed.

5. **Rebuild the 100%-pass suites from scratch with real rendering assertions.** The following reported 100% pass but are actually coverage theatre:
   - Layer2 (61/61) — missing ~30 plan tests; bank `+1` transform untested; palette-RGB transparency (historical bug class) untested; port 0x123B mapping zero coverage
   - Sprites (48/48) — 8+ tautologies; zero rendering tests; anchor/relative composition untested
   - Copper (69/69) — mode 10 semantics wrong in plan; self-modifying copper (EDG-03/04) untested; arbitration stubbed
   - Compositor (74/74) — anti-tests lock in wrong blend-mode behaviour; RGB transparency uses ARGB alpha instead of VHDL palette compare; ~70% of plan missing
   - I/O Port Dispatch (78/78) — tests container class, not real peripheral registrations; libz80 C-only regression untested; NR 0x82-0x89 enable bits untested
   - Input (71/71) — only keyboard membrane tested; 6 of 8 joystick categories empty; Kempston bit map, JMODE-02 example, MD-01 expected all have plan arithmetic errors

   For each: retract the 100% status, build an independent VHDL-derived oracle, add boundary/negative tests, and measure honest pass rate.
