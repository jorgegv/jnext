- Make debugger (and debug window) optional: add a debug button on the toolbar, and Enable/Disable Debugger options in the Debug menu.
- When the debugger is disabled, the emulator window should be exactly equal as when no debugger is configured (debugger hidden completely)
- There should be NO PERFORMANCE IMPACT when Debugger is disabled
- The Debugger panels should show on an aditional window and only when enabled. CLosing the debugger window should disable debugger and continue execution at normal speed
- The CPU registers panel shows 4 groups and the "Flags and State" panel in a vertical layout. The first 4 groups should be shown horizontally in parallel, so that we can see all registers in a single panel. then the Flags and state panel below.
- The disassembly panel should be shown as wel below the registers panel, but at 2/3 the current vertical size (we don't need so many lines of context). Also, the line that is currently executing should be shown in the middle of the panel, not on top. This way we can see the instructions before and after the current one.
- The memory view at the bottom should be taller, it should at least show 16 lines of memory bytes.
- Also: the debugger should have its own toolbar with the the Step Into, Step Over, Step Out buttons, together with Break and Continue buttons. You can use buttons with the literal texts.
- Remove step into/over/out buttons from main toolbar, they belong in debug toolbar
- Remove all controls from the panels, theu should have a static layout (no maximize/close minibuttons). Also add some light gray lines separating the panels
- Make the 4 register groups in the register panel occupy evenly the whole panel width and draw some light vertical lines between the groups to separate them
- Debugger window should show sticked to the right of the emulator window, and aligned to the top. Both windows should move together when the debugger is shown.
- If the main window is closed, the debugger window should be closed also and the emulator end
