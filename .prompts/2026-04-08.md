## Task 1: several fixes

- Video debugger panel redesign
  - Redesigned VideoPanel UI to match CPU panel style: bold headings + monospace values on the same row
  - Headings padded to same width (13 chars) so value columns align across all rows
  - Layers and Priority rows: items separated by extra spacing, green/gray coloring for active state
  - HC/VC: fixed-width labels (`fontMetrics().horizontalAdvance("000") + 4`) formatted as `%3d`
  - Horizontal line separator at the bottom only

- Fix HC counter range
  - Was dividing by 2 (0..911); corrected to divide by 4 (pixel clock = 28MHz/4 = 7MHz → 1 pixel = 4 master cycles), giving 0..455

- Fix VC=0 when Break is pressed
  - Root cause: `run_frame()` advances `frame_cycle_` to `frame_end` at end of each frame.  When Break fires after the frame completes, `clock_.get() - frame_cycle_ ≈ 0` → VC=0.
  - Fix: save `last_frame_vc_` / `last_frame_hc_` just before `frame_cycle_` advances.
  - `snapshot_raster()` detects "between frames" condition (`elapsed < MASTER_CYCLES_PER_LINE`) and uses the saved values instead of computing from a near-zero elapsed.
