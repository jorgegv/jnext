## Housekeeping / small follow-ups (backlog)

- [ ] **`Emulator::prev_nmi_generate_n_` reset gap** — constructor-init
  only; not reset in `Emulator::reset()` / `init()`. Re-initialising a
  long-lived Emulator after a prior NMI can mask the next falling
  edge. Surfaced by NMI Phase 3 integration test 2026-04-24. Fix: add
  `prev_nmi_generate_n_ = true;` to `Emulator::reset()` (and `init()`
  if separate). Tiny one-liner + a regression test.
- [ ] **Rename `Renderer::ula_over_flags_[x]` → `tm_pixel_below_[x]`**.
  The current name is misleading: the bool semantically means
  "tilemap pixel is below ULA" (per VHDL `tm_pixel_below_2`), not
  "ULA pixel is over TM". Flagged by the UDIS-03 Phase 0 Q1
  resolution and deferred from Phase 1 scope. Cosmetic rename;
  touches all call sites in `src/video/renderer.cpp`.
